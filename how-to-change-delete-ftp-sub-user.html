How to Change or Delete an FTP Sub-User in CloudHoste Panel

In CloudHoste Panel, you manage FTP sub-users from the FTP Accounts page. To rotate a password, open the row’s Actions menu and choose Change password. To remove access, tick the account’s checkbox and click Delete Selected, then confirm.

NOTE UP FRONT: Deleting a sub-user removes their login access but leaves the files in their home directory untouched. Cleanup of files is a separate step.

Overview: Who This is For & What Problem This Solves

This guide is designed for website owners who previously provided a contractor, freelancer, or external agency with their own FTP login and now need to rotate that password or revoke access entirely. Common triggers include:

  • Project Wrapping: Revoking credentials post-delivery to enforce security boundaries.
  • Credential Leaks: Instantly securing the server after developers inadvertently share configuration files.
  • Routine Rotation: Complying with monthly or quarterly security auditing policies.

By managing individual sub-users, you avoid sharing your main account credentials, eliminating the risk of having to re-configure all of your local tools and scripts if a password needs to be changed.

How CloudHoste Panel Solves This

CloudHoste Panel treats each FTP sub-user as a completely isolated entity listed in the FTP Accounts directory. Each account row provides per-row actions:

  • Password Rotation: Modifies only the selected sub-user's credentials. Your main account log-ins remain untouched, and the change takes effect immediately on the next connection attempt.
  • Access Revocation: Selecting the account and utilizing the bulk Delete Selected function removes the sub-user records instantly. Since there is no edit-name function, renaming a user is solved by deleting and recreating the account.

Before You Start Checklist

Make sure you have configured:

Login Access

Ensure you have admin-level control over the CloudHoste Panel account managing the subscription.

Namespaced Accounts

Identify the target account. Active sub-users follow the format username_subuser namespace prefix.

Data Backup

Retrieve contractor-uploaded files before revoking credentials. The files stay on the server, but their access vanishes.

Temporary Pausing

To temporarily suspend access instead of deleting permanently, rotate the password to a secret key instead.

Step-by-Step Revocation & Rotation Guide

Step 1
Locate FTP Section

Log in to your CloudHoste Panel control panel, search for FTP, and click on FTP Accounts. Identify the correct sub-user row.

Step 2
Access Action Menu

Click on the Actions menu on the right. You will see options to Change password or Connect an FTP client.

Step 3
Rotate Credentials

Choose Change password, enter your strong new credentials in the modal dialog, and click save. The change propagates instantly.

Step 4
Delete Selected Accounts

To delete access, tick the checkbox next to the sub-user's row, select Delete Selected, and click YES on the confirmation prompt.

Troubleshooting & Limitations

Symptom / Limitation Likely Cause Action Plan
Contractor connects after password change Active FTP sessions may stay open until terminated. Permanently delete the sub-user account to force immediate drop of connections.
Can't find edit-name option CloudHoste Panel FTP schema doesn't support direct renaming. Delete the old sub-user login and provision a new one with the target naming configuration.
Storage files remain after user deletion Authentication configuration is decoupled from the directory storage path. Navigate to File Manager and manually delete the user's home directory.
Wrong sub-user target selected Namespaces look identical in high volume listings. Cancel confirmation modals immediately and re-evaluate namespaced usernames.

When to Use / When Not to Use

Use This Process When Avoid/Skip This Process When
Contractor project has concluded. You only need a brief pause (change password to secure key instead).
Suspecting user credential leak. You want files destroyed along with credentials (must perform file cleanup separately).
Doing standard credential rotation audits. You want a different username (delete and recreate).
Enforcing credential isolation boundaries. You want to reset main admin billing or shell passwords.
