How to Create a Staging Site

A staging site is a testbed where you can check whether any changes you’ve planned for your website work as expected.

For example, if you’re thinking of installing a plugin on a WordPress project, you’ll likely want to confirm that it doesn’t create any compatibility issues with other themes and add-ons. With a staging site, you have an identical environment as your main project, so you can test if everything works fine before pushing the new plugin to production.

Let’s have a look at the different options for creating a staging environment.

Creating a Staging Site via the WordPress Manager

IMPORTANT: Before creating a staging area, make sure to back up your entire website data. This is essential to avoid any possible issues.

If you use one of CloudHoste’s CloudHoste Panel managed VPS plans, you have an easy way of creating a staging environment for your WordPress project. The unique WordPress Manager is equipped with this sort of functionality, and we’ll now show you just how easy it is to take advantage of it.

Step 1
Access CloudHoste Panel

Log into your CloudHoste Panel control panel and navigate to the WordPress Manager located under the Software section.

Step 2
Select Actions

All of your WordPress installations under the specific account will be listed there. Locate the target installation and click on the Actions button.

Step 3
Select Stage Option

From the actions list, select the Stage option to initiate the cloning configuration process.

Step 4
Configure Destination

A pop-up window will be shown from which you might select to either generate the staging environment in a pre-selected subdomain (e.g., wordpress.domain.com) or create a new subdomain with the desired name.

Click on the blue Stage button to finish the installation. Now that you have a staging area, you can test any changes without disrupting the operation of your live site.

Creating a Staging Environment for a Non-WP Site

Cloning a WordPress site is easy because you’ve got plenty of tools that can automate the process for you. However, if you have a custom website or use another CMS, these tools may not be at your disposal. If that’s the case, you’ll need to manually clone your website. It's not as instantaneous as one-click utilities, but the process is straightforward:

1
Copy Your Site's Files

Because FTP clients like FileZilla present limitations when transferring massive amounts of small files, it is best to use your hosting control panel's File Manager. Navigate to your site's files, select all files and folders, and use the Copy button to save them to the server's clipboard.

2
Create the Staging Folder

If you are going to use a subfolder (e.g., yourdomain.com/stage), you can create the folder directly from the File Manager. If you want to host the clone under a subdomain (e.g., dev.yourdomain.com), navigate to subdomains settings and configure the folder root.

3
Paste Files into Destination

Paste the copied files and folders into the new staging folder. The wait time depends on your site’s size and server resources.

4
Export Your Live Database

Create a separate database for the clone site to isolate its changes. Open phpMyAdmin, select the correct database, click Export, and save the data to your local computer as an SQL file.

5
Create a New MySQL Database and User

Create an empty database and user for your staging site. CloudHoste Panel users can easily create a database and user via the MySQL Databases tool available on the homepage.

6
Import Data into the New Database

Go back to phpMyAdmin, select the new database, click Import, select the SQL file exported in Step 4, and run the import.

7
Configure Database Settings

Finally, update your staging site's configuration file (e.g., wp-config.php for WordPress or config files for other CMSs) with the credentials of the new database and staging URL.

If you need any additional assistance setting up your staging environment, do not hesitate to contact our Support Team 24/7 through live chat, email, or ticketing.
